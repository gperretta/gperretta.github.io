(Re)design for HIG!

A first attempt at app redesign. Case study: UnicoCampania iOS App.

This two-weeks project was mainly aimed at practicing more with the iOS Human Interface Guidelines, exploring SwiftUI and the use of native components.
After a brief study of the app user-flow, I've implemented the UI of the main call to actions about tickets: check/validate your active tickets and buy a new one.
The final redesigned app contains:
- a horizontal scrollview to display multiple active tickets,
- a list for the three paths to buy a new ticket and a full-screen modal for ticket validation,
- a unique QR code, that can be reloaded both manually by the user and automatically with a timer.
See the project repository on GitHub
Sprout
